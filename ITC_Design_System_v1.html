ITC Design System
This document is the single reference for all ITC website pages. Every practice area page, landing page, and content page should use only the colors, fonts, components, and patterns shown here. If it is not in this guide, ask before building it.
Purpose of this document All five practice area pages must look like they belong to the same firm, built by the same team. This guide ensures visual consistency across every page regardless of who builds it or when. When in doubt: match what is in this guide exactly. Do not invent new colors, fonts, or component patterns.

Typography rules Playfair Display is for headlines, framing sentences, card titles, phase titles, and the positioning statement only. Never use Playfair for body copy. Never use it at weights other than 500 or 700. Source Sans 3 weight 300 is the default for all body text — it reads lighter and more confident than 400. Use 600 only for button labels and nav items.

What Not to Do
These patterns appeared in earlier ITC content. Do not reproduce them on any new page.
🚫 Decorative borders and scroll ornaments Do not use decorative horizontal rules, scroll flourishes, ornamental dividers, or clip-art style section separators. Section separation is handled by background color alternation and 1px solid #E0E0DC borders only.
🚫 Flat stat tables Do not put statistics in HTML tables or flat rows. All statistics use the Stat Card component — white card, rounded corners, Playfair number, source attribution. See Components section above.
🚫 Long case study narratives on practice area pages Case studies belong on a dedicated Cases page or as linked content. Practice area pages use the two-panel Winning Signal Card component for outcome evidence. Full narratives bury the conversion architecture.
🚫 Unattributed statistics Every stat on the page must have a source in the stat-source line. "Source: Perplexity" is not a citation. All stats on ITC pages come from the validated research library: Gartner, McKinsey, Panorama, Deloitte, Gallup, WEF, Accenture, HBR, Prosci. See the companion document for the full source appendix.
🚫 Font substitutions Do not substitute Georgia for Playfair Display. Do not use system fonts for headings. Do not use Arial or Helvetica for body copy. If Google Fonts are not loading, check the link tag in the theme header. The font stack is Playfair Display + Source Sans 3 + Source Code Pro — no substitutions.
🚫 Gold as a primary action color Gold (#D4A439) is used only for borders on the positioning statement, snapshot sample item, and winning card right panel. It is never used for buttons, links, or interactive elements. Teal is the action color.
🚫 Fear-based or alarm headlines Headlines like "Don't Become a Negative Statistic" do not match ITC's voice. Headlines and section titles on ITC pages are declarative, peer-to-peer, and outcome-named. See the framing sentence and Areas of Focus components as the voice model.
Divi Implementation Notes
Specific notes for building these pages in Divi 5 on WordPress. Platform: itc.pro hosted by Training Lion (Robert Staddon).
CRITICAL: H1 and page title Leave the Divi page title field BLANK. The H1 renders inside the Fullwidth Header module "Title" field. If both are populated, Google sees two H1 tags — this breaks SEO. The H1 is the SEO carrier for the page. It must be unique and correct.
Content max-width in Divi Set row max-width to 960px in Divi Row Settings. All content is centered within this column. Section backgrounds fill 100% width. Use 24px horizontal padding on sections (Divi Section Settings: padding left/right 24px).
Custom CSS in Divi Use Divi Theme Options → Custom CSS OR the Divi Module Settings → Custom CSS tab. CSS variables from this guide work in Divi custom CSS fields. Example: color: var(--teal); background: var(--navy); Add the :root variable block to Divi Theme Options → Custom CSS at the top level.
Google Fonts Add the font link tag to Divi Theme Options → Integration → Add code to the <head> of your blog: <link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,500;0,700;1,500&family=Source+Sans+3:ital,wght@0,300;0,400;0,600;1,400&family=Source+Code+Pro:wght@400&display=swap" rel="stylesheet">
SEO: Rank Math SEO Pro Already installed on itc.pro. For each practice area page: — Rank Math sidebar → Focus Keyword: enter primary keyword — Rank Math → Snippet Editor: set meta title and meta description — Rank Math → Schema: add LocalBusiness (site-wide), FAQPage (Snapshot section), BreadcrumbList — Enable BreadcrumbList: Rank Math → General Settings → Breadcrumbs → On Do NOT use Yoast or any other SEO plugin alongside Rank Math — conflicts will occur.
